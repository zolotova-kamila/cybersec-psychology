# 📄 Инструкция: Как добавлять статьи на сайт

## Структура раздела "Материалы"

Раздел разделён на 3 категории:
1. **Статьи** — с превью-картинками (как на Medium)
2. **Презентации** — PDF для просмотра онлайн
3. **Документы** — DOCX для просмотра (не для скачивания)

---

## 📝 Процесс добавления новой статьи

### Шаг 1: Создание HTML-файла статьи

1. Создай файл `article-[название].html` в корневой папке
2. Используй шаблон существующей статьи
3. Обязательно замени:
   - `title` в `<head>`
   - `meta description`
   - `og:title`, `og:description`, `og:image`
   - Заголовок H1
   - Контент статьи
   - Дату публикации в Schema.org

### Шаг 2: Подготовка картинки-превью

**Требования:**
- Размер: 800x400 пикселей (2:1)
- Формат: JPG или WebP
- Размер файла: до 200 KB
- Название: `article-[название-статьи].jpg`
- Сохранить в папку `images/`

### Шаг 3: Добавление в раздел "Материалы"

Открой `resources.html` и найди раздел `<!-- Статьи -->`.

Добавь новую карточку (шаблон в файле resources.html)

### Шаг 4: Обновление sitemap.xml

Добавь новую страницу в `sitemap.xml` с приоритетом 0.7

### Шаг 5: Публикация

1. git add -A && git commit -m "Add new article: [название]"
2. git push origin main
3. Загрузи на хостинг через lftp

---

## ✅ Чек-лист перед публикацией

- [ ] HTML-файл создан и проверен
- [ ] Картинка-превью подготовлена (800x400)
- [ ] Карточка добавлена в resources.html
- [ ] Sitemap обновлён
- [ ] Все файлы загружены на хостинг
- [ ] Страница проверена на работоспособность

---

## 🚫 Важно: НЕЛЬЗЯ делать

1. **Не используй кнопки "Скачать"** для статей и документов
2. **Не добавляй атрибут `download`** к ссылкам
3. **Не размещай статьи без превью-картинок**
4. **Не забудь про Schema.org разметку**
